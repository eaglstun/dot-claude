# wan-video/wan-2.7-i2v

Model page: https://replicate.com/wan-video/wan-2.7-i2v

Alibaba's **Wan 2.7 image-to-video** — animates a still image (or extends a video clip) into a 2–15s video at up to 1080p, with auto-generated audio if you don't supply one. Four generation modes share a single endpoint; the mode is determined by which optional inputs you set.

This is a companion to `wan-video/wan-2.7-videoedit` (prompt-driven editing of existing video) — use i2v when you're creating motion from a still, and videoedit when you're modifying an existing clip.

## Modes (inferred from inputs)

| Mode                     | How to trigger                                                           |
| ------------------------ | ------------------------------------------------------------------------ |
| **Image-to-video**       | Set `first_frame` + `prompt`.                                            |
| **First-and-last-frame** | Set `first_frame` + `last_frame` + `prompt` — model morphs between them. |
| **Clip continuation**    | Set `first_clip` + `prompt` — extends from the end of the clip.          |
| **Audio-synchronized**   | Any of the above + `audio`. Output syncs to the audio track.             |

`first_clip` **cannot be combined with** `first_frame`.

## Input schema

| Field                     | Type         | Required                        | Default   | Description                                                                                 |
| ------------------------- | ------------ | ------------------------------- | --------- | ------------------------------------------------------------------------------------------- |
| `first_frame`             | string (URI) | (one of first_frame/first_clip) | —         | Starting image. jpg/png/bmp/webp, ≤20MB.                                                    |
| `last_frame`              | string (URI) |                                 | —         | End image for first-and-last-frame mode. Requires `first_frame`. jpg/png/bmp/webp, ≤20MB.   |
| `first_clip`              | string (URI) | (one of first_frame/first_clip) | —         | Video clip to continue from. mp4/mov, 2–10s, ≤100MB. Mutually exclusive with `first_frame`. |
| `audio`                   | string (URI) |                                 | —         | Audio file (wav/mp3, 3–30s, ≤15MB). If omitted, model auto-generates matching audio.        |
| `prompt`                  | string       |                                 | `""`      | Describes the desired motion/action. "Describe the motion" beats "describe the scene" here. |
| `negative_prompt`         | string       |                                 | `""`      | What to avoid.                                                                              |
| `duration`                | integer      |                                 | `5`       | Seconds, range 2–15.                                                                        |
| `resolution`              | enum         |                                 | `"1080p"` | One of `720p`, `1080p`.                                                                     |
| `enable_prompt_expansion` | boolean      |                                 | `true`    | Auto-expands short prompts for better results. Small latency cost.                          |
| `seed`                    | integer      |                                 | random    | For reproducibility. Range: 0–2147483647.                                                   |

Schema quirk: no fields are marked `required` in OpenAPI, but practically you must provide either `first_frame` **or** `first_clip`. Supplying neither will error.

Local files for `first_frame`, `last_frame`, `first_clip`, `audio` are auto-uploaded by `run_model.py`.

## Output

Single URI to the generated video (MP4). Saved as `wan-video_wan-2.7-i2v_0.mp4`.

## Audio behavior

This is distinctive: **audio is generated by default**. If you don't pass `audio`, the model synthesizes a matching track (ambient sound, effects, sometimes music). This is different from most i2v models that output silent video. If you want silent video, you'll need to strip audio post-hoc with ffmpeg:

```bash
ffmpeg -i wan_output.mp4 -c copy -an silent.mp4
```

If you want to **drive** the video with a specific audio track (music video, lipsync-style), pass `audio` and the model will sync visuals to it.

## Examples

**Simple image-to-video:**

```bash
python scripts/run_model.py wan-video/wan-2.7-i2v \
    --input '{
      "first_frame": "./landscape.jpg",
      "prompt": "slow dolly-in, clouds drift across the sky, leaves rustle in the foreground",
      "duration": 5,
      "resolution": "1080p"
    }' \
    --output ./out/
```

**First-and-last-frame morph:**

```bash
python scripts/run_model.py wan-video/wan-2.7-i2v \
    --input '{
      "first_frame": "./day.jpg",
      "last_frame": "./night.jpg",
      "prompt": "smooth transition from day to night, stars fade in, long shadows lengthen",
      "duration": 6
    }' \
    --output ./out/
```

**Extend an existing clip:**

```bash
python scripts/run_model.py wan-video/wan-2.7-i2v \
    --input '{
      "first_clip": "./intro.mp4",
      "prompt": "camera continues to track forward, subject turns and walks into the distance",
      "duration": 8
    }' \
    --output ./out/
```

**Audio-synced music video:**

```bash
python scripts/run_model.py wan-video/wan-2.7-i2v \
    --input '{
      "first_frame": "./performer.jpg",
      "audio": "./track.mp3",
      "prompt": "performer moves to the beat, stage lights pulse in sync with the music",
      "duration": 15
    }' \
    --output ./out/
```

## Tips

- **Describe the motion, not the scene.** The image already says what's in the frame. Use the prompt for _what changes_: camera direction, subject action, environmental shifts.
- **Keep under 10s for best quality.** 10–15s runs are supported but degrade more than the 2–8s sweet spot.
- **Prompt expansion is usually worth it.** Short, terse prompts benefit noticeably. Turn off only when you've hand-tuned a long, specific prompt.
- **Negative prompts** work for common failure modes: `"blurry, distorted faces, warping, morphing, extra limbs"`.

## Gotchas

- `first_frame` and `first_clip` are **mutually exclusive** — supplying both errors out.
- `last_frame` requires `first_frame`; pairing `last_frame` with `first_clip` isn't supported.
- **Audio is generated by default** — strip it post-hoc if you need a silent track.
- Complex multi-character scenes may have identity drift across frames; consider single-subject shots.
- File size limits: images ≤20MB, clips ≤100MB, audio ≤15MB.
