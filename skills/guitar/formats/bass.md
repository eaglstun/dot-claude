# bass.md format

Bass chart in **lead-sheet flow**: one document the player reads top to bottom during
the song. It mirrors `lead.md` exactly — same sections, same chord rows, same lyric
cue lines — with bass-specific info inline where it happens. No separate
quick-reference table and no per-chord tab library: the song order IS the chart, and
a single compact Positions table up top covers where every note lives.

Generated by `scripts/gen_chart.py --only bass` from the song's `chart.py` +
`lead.md`. Do not hand-edit the output — put hand-written content in `chart.py`'s
`BASS_CUES` and regenerate.

```markdown
# Song Title — Bass

Standard tuning, low→high: **E A D G**. Play the chord root; on slash chords (`X/Y`) play the slash note `Y` — inline `bass:` cues call these out. Positions written `String-fret` (`A-5` = A string, 5th fret); the table below shows where every note in the song lives.

## Positions

| Note | Low (E/A)  | Octave up (D/G) |
| ---- | ---------- | --------------- |
| F♯   | E-2 · A-9  | D-4 · G-11      |
| D    | A-5 · E-10 | D-open · G-7    |

## Intro — bars 1-18

| F#m //// | F#m //// | (2 bars)

| D //// | D //// | D //// | D //// |

## Chorus 1 — bars 27-40

| Eadd9 //// | Eadd9 //// | Dmaj7 //// | Dmaj7 //// |
Rolling down hills, never looking back
_Dmaj7 — Lush landing after the build. Give it room; let it arrive._

## Break 1 — bars 41-48

| A6/F# //// | A6/F# //// | Dmaj7/F# //// | Dmaj7/F# //// |
_bass: F♯ (E-2) throughout_
```

## Format rules

- Title: `# Song Title — Bass`
- Preamble: the standard tuning/notation paragraph (one source line — flow-file PDFs
  render soft breaks for real)
- `## Positions` table: one row per unique bass target note (chord root, or slash
  bass for `X/Y` chords), in order of first appearance in the song. Columns: pretty
  note name (`F♯` not `F#`), low positions (E/A strings), octave-up positions (D/G
  strings), positions `·`-separated, ascending fret, `open` for fret 0
- Song body: sections, chord rows, and lyric lines copied **verbatim** from
  `lead.md` — same headers with bar ranges, same row layout, same `(N bars)`
  annotations
- Inline bass cues, italic, on the line right after the row's lyric (or right after
  the row when instrumental):
  - **Auto slash cues** — any row containing slash chords gets `_bass: ..._`:
    `_bass: F♯ (E-2) throughout_` when every chord in the row targets the same
    slash note, otherwise `_bass: A7/D → D (A-5) · C#m/A → A (A-open)_` per chord.
    Repeated identical cues within a section are emitted once
  - **Hand feel cues** — from `chart.py`'s `BASS_CUES` dict: rendered as
    `_<key> — <text>_` under the FIRST `lead.md` row matching the key. Keys are a
    chord name (`"Dmaj7"`) or a `·`-joined chord sequence
    (`"A7/D · F#7sus4/D"`) that must appear, in order, in a row. A row with a hand
    cue skips its auto slash cue (the hand cue covers it). A key matching no row is
    a generation error, not a silent drop
- No tuning/fret reference table in the file (lives in `references/tuning.md`), no
  quick-reference table, no per-chord tab blocks

## BASS_CUES authoring (in the song's chart.py)

```python
BASS_CUES = {
    "Dmaj7": "Lush landing after the build. Root on D, but give it room.",
    "A7/D · F#7sus4/D": "Stay on D through the whole vamp while the chords "
                        "above you shift.",
}
```

Write cues for: slash chords where the note choice needs a why, walkups/walkdowns
the bass leads, pedal-point vamps, and feel guidance (space, tension, resolution
targets). Root-following rows need no cue — silence means "play the root".
